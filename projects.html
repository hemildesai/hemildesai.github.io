Twitter Connections

Twitter Connections which matter!

ChallengePost
Youtube

Connections is an Android app that helps people help each other. With Connections, the user can use social networking to find people who either need rides, books, or some other kind of help, or are willing to sell, carpool, or offer some other kind of help. Connections does this with the help of Hashtags in Twitter. The user logs in with Twitter inside the app, and then can look for people giving services, and then contact them via Twitter. If the user needs services like mentioned above, he can select from the inbuilt services or create a new service, and then Connections posts it on Twitter once the user has finished entering the fields required for the certain service. The target users are mostly College students but because of the versatility of the app everyone can use it.

Learning Experience!

Android SDK + Android Studio
Integrating the Twitter Fabric API into the application
Using Twitter Hashtags to create search queries
Using Twitter Feed of a user as a database to store requests
Software Development Lifecycle and Teamwork Experience

Challenges Faced!

Integrating and using Fabric API for the first time
Integrating front end design with the back end
Creating the appropriate search query using hashtags
Coordinating and Collaborating among the team in a given time constraint

The Social Market

An easy way for students to buy and sell stuff

Github

HackMIT was my first hackathon and it was a wonderful learning experience. We came up with this idea to create a simplified platform for students to trade things while maintaining authentication. The purpose of the web application is for students to list things to sell and provide a basic interface for other students to search for those things. To add security, it is important that the students provide their valid .edu acoount. This account is then verified and students can access the web application. This way students at a university are matched through the website and then they can easily meet to complete the transaction. The website was built using Facebook + Parse as the database.

The Safewalk Simulator Project

A three part project - both academic and extra-curricular

Part 1
Part 2
Part 3

Safewalk is a system in Purdue that allows requesters who feel unsafe to request volunteers to help them safely reach their destination. The safewalk project was a three part project that I made as one of my very first learning experiences. The first part was to implement an Android Application that allows users to request volunteers much like uber. It also parsed messages from the server and displayed the log of all activities since the server started. The second part was to implement a Safewalk Monitor which was a database storing the entire information of the system and a GUI used to monitor the system. A controller was used to communicate between the Monitor and the database. The third part was to implement an AI client for a particular volunteer. The client helps the volunteer to select the best requestors in order to score the highest points based on a scoring system. The client was tested on time constrained rounds in a competition and my algorithm was among the top ones.

Learning Experience!

Introduction and initial exposure to the Android SDK
Implementing a database and monitor using Model-View-Controller paradigm in java
Object Oriented Programming principles including classes, interfaces, inheritance, abstraction, etc.
Data structures including hash table, array lists and hash sets
Implementing the mathematical algorithm using the coordinate system into the AI client

Challenges Faced!

Setting up and using the android environment for the first time
Communicating between the database and the GUI and parsing messages from the server
Algroithm testing and implementation
Using the graphics library to implement the GUI
Yep, that's the Safewalk project
